   OlymarsInformation Attribute
Overview
This class, which derives from the System.Attribute class, contains all the information needed to be synchronized between the database schema and the generated code. If the assembly is compiled with either the /define:OLYMARS_DEBUG or the /define:OLYMARS_ATTRIBUTE flags, this class will be included in all the classes that make a database access.

Furthermore, if the assembly was compiled with /define:OLYMARS_DEBUG flag, a check is made each time you use one of the class. This call verifies that the current SQL object version is in sync with the one stored in the attribute of the class. If the version numbers are not in sync anymore, an exception is thrown.

However, if you know that your code is not in sync anymore with the database schema but you want to be able to run the code anyway, you can do so by adding a new key in the configuration file:
<?xml version="1.0" encoding="utf-8" ?>
<configuration>
  <appSettings>
    <add key="OlymarsDebugCheck" value="False" />
  </appSettings>
</configuration>
