Overview
This class must be used in conjunction with the [Bob.DataClasses.StoredProcedures.spIU_Customers] class. These classes allow you to execute the [spIU_Customers] stored procedure.

Basically, you use the "Parameter" class to supply the needed parameters to the stored procedure. Then, you use the "StoredProcedure" class to actually execute the stored procedure. Once executed, you can retrieve back the ReturnValue and all the output parameters by using again the "Parameter" class.

This class inherits from System.MarshalByRefObject and implements the following interfaces:
  • System.IDisposable
  • Bob.DataClasses.IParameter

Sets the connection string to be used against the SQL Server database.

connectionString: A valid connection string to the database.

If a blank connection string is supplied, the class will search for:

  First an entry in the configuration file:
    <configuration>
     <appSettings>
      <add key="Bob ConnectionString" value="..." />
     </appSettings>
    </configuration>

  Then the following registry entry:
    -Key: HKEY_LOCAL_MACHINE\SOFTWARE\ConnectionStrings\Bob
    -String Value: ConnectionString

A helper function that can build this connection string for you is available. See the BuildConnectionString function description.
