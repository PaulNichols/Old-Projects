Overview
This class is located in Bob.Windows namespace. It allows you to quickly fill on TreeView with Bob database table content.

All methods of this class are static.

To be able to use this class, you need to call the SetUpConnection once for good. Then, you can call any of the Fill_<TableName> method to fill out a TreeNodeCollection object with the <TableName> table content.

Once you have populated a TreeNodeCollection, this collection contains Bob.Windows.TreeNodeFactory.TableCustomTreeNode based nodes. For each of this node, you can query information such as:
  • Id: primary key of the record
  • Table: returns the table from which this record was loaded (see Bob.Windows.TableList enumeration for more information).

Sets the connection string to be used against the SQL Server database.

connectionString: A valid connection string to the database.

If a blank connection string is supplied, the class will search for:

  First an entry in the configuration file:
    <configuration>
     <appSettings>
      <add key="Bob ConnectionString" value="..." />
     </appSettings>
    </configuration>

  Then the following registry entry:
    -Key: HKEY_LOCAL_MACHINE\SOFTWARE\ConnectionStrings\Bob
    -String Value: ConnectionString

A helper function that can build this connection string for you is available. See the BuildConnectionString function description.
