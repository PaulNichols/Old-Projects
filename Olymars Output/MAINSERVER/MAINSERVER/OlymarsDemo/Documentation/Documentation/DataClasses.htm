OlymarsDemo.DataClasses namespace
   Overview
Overview
The OlymarsDemo.DataClasses namespace provides a set of classes that can make all the necessary data access calls to the SQL Server database stored procedures.

When you call a stored procedure, you usually perform 4 tasks:
  • 1- Affect the stored procedure parameters before the call
  • 2- Call the stored procedure itself
  • 3- Retrieve back the stored procedure return value and all the output parameters
  • 4- If provided, retrieve back a resultset (or an XML stream)

In order to perform those 4 tasks, two types of classes are provided:
  • "Parameter" type of class: allows you to perform task #1 and #3
  • "StoredProcedure" type of class: allows you to perform task #2 and #4

When you want to execute a given stored procedure, search for the 2 classes located in the OlymarsDemo.DataClasses.Parameters and OlymarsDemo.DataClasses.StoredProcedures which names are the names of your stored procedures. Whit these two types of class, you will able to successfully call any stored procedure of your database.
Parameter
Before the stored procedure call, this type of classes allows you to supply the following information:
  • The connection information: how do you want to connect to your database
  • Parameters values: what value do you want to affect to each of your stored procedure parameter
After the stored procedure call, this type of classes allows you to retrieve the following information:
  • Parameters values: Return Value and all the Output parameters
  • Exception information: information on exceptions that were thrown
StoredProcedure
This type of classes allows to actually make the call to your stored procedure.

If the stored procedure returns back a resultset (or an XML stream), this class will also give you the ability to get back the data using various ways, such as getting back a System.Data.DataSet, System.Data.SqlClient.SqlDataReader, ...
